DDS

The DDS entity is returned as the payload of a message whose Content-Type header MUST be text/plain. The body of the response contains text which holds all of the variables, their names and datatypes.

The DDS is a textual description of the variables and their names and types that compose the entire data set. The data set descriptor syntax is similar too the variable declaration/definition syntax of C and C++. A variable that is a member of one of the base type classes is declared by by writing the class name followed by the variable name. The type constructor classes are declared using C's brace notation.

dds-doc = "dataset" "{" *type-decl "}" id ";"
type-decl = simple-decl | list-decl | array-decl
| structure-decl | sequence-decl | grid-decl

The dataset keyword has the same syntactic function as structure but is used for the specific job of enclosing the entire dataset even when it does not technically need an enclosing element (because at the outermost level it is a single element such as a structure or sequence).

An example DDS is shown in Figure *. 

dataset {
   int catalog_number;
   sequence {
      string experimenter;
      int32 time;
      structure {
         float64 latitude;
         float64 longitude;
      } location;
      sequence {
         float depth;
         float temperature;
      } xbt;
   } casts;
} data;
