Errors and User Interactions

  • Error objects are generated locally or remotely and rendered either using a GUI or TTY, depending on the build- and run-time configuration settings.

  • Error objects can contain embedded programs. These programs should provide users with a means to correct the error (e.g., edit and resubmit an incorrect URL)

  • Error objects currently use Tcl/Tk but contain hooks for Java as well.

  • The software used to build the error processing system can be used to communicate other types of information to users; the progress indicator uses Tcl/Tk and an embedded interpreter too
