Searching for data

Users must be able to:
  • Locate different types of data
  • Locate different datum within sets of data
  • `Converse' with the location mechanism






Searching for data means much more than making a request to a relational database and processing the resultant data. Users must be able to engage in a back-and-forth conversation to find interesting collections of data. Furthermore, they must be able to use these data in new ways.
