Characterization of a Dataset

The DAP characterizes a dataset as a collection of variables and a collection of attributes. Variables contain the names, datatypes and vlaues of the information in the dataset. A dataset may also have associated with each variable zero or more attributes. Each attribute has a name, datatype and value.

A dataset's variables are described by two of the three basic responses. The DDS response holds the name and datatype of each variable in the dataset. It is essentially a declaration of the variables in the dataset. Sections * and * describe the diffent types of variables in the DAP.

The DAS response holds all of the attribute information for the dataset. Attributes are a way to bind extra information to a variable and to an entire dataset. While the datatypes supported for attributes are limited to simple types and vectors of simple types, the actual collection of attributes is completely unconstrained. Furthermore, the DAP does not have any standard set of attributes it requires for variables or datasets. See Section * for more information about the DAS response.

Both the DDS and DAS responses describe the dataset and the variables it contains. To request data, a client asks a server for a DataDDS object. The DataDDS class is a subclass of DDS and as such it holds information about the names and types of variables in the data source. In addition, a DataDDS object holds values for those variables. See Section * for information about encoding values in a DataDDS response.
